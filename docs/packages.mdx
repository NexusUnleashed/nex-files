# Packages

Choose a focused package or combine them into a complete Nexus setup.

## Version 4

### [nexSys4](/docs/nexsys/introduction)

Character state and curing automation. Tracks afflictions, defences, balances,
limbs, queues, precache, and server-side curing configuration.

### [nexMap4](/docs/nexmap/introduction)

Mapping, pathing, travel, and room intelligence. Public documentation is a
work in progress.

### [nexGui4](/docs/nexgui/introduction)

The visual workspace for package state, streams, tables, and gameplay panels.
Public documentation is a work in progress.

### [nexBash4](/docs/nexbash/introduction) {#nexbash4}

Hunting and battlerage decision support. Coordinates area routes, target
selection, class strategies, safety automations, and run scoring on top of
nexSys4 state.

## Combat intelligence

### [insight](./insight/introduction.mdx)

Probabilistic target tracking for afflictions, defences, balances, limbs, and
combat state.

## Foundations

### [eventStream](./eventStream/introduction.mdx)

The shared client-side event layer and GMCP event bridge.

### [nexAction](./nexAction/introduction.mdx)

A lightweight trigger and alias engine used by packages and custom scripts.

## Experimental packages

Early work lives in [Labs](./labs/index.mdx). Labs projects are useful to
explore, but their behavior and public interfaces may change without notice.

### [nexGear](./labs/nexgear.mdx)

An alpha equipment and gear-set manager with searchable inventory, custom set
assignment, and locally persisted selections.

### [nexWorld](./labs/nexworld.mdx)

An early package exploration that does not yet have a stable public scope or
installation path.
