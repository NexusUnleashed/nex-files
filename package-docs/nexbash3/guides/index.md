# Guides

Use these pages when you want to accomplish a task or understand how a nexBash4
feature behaves.

## Running and configuring

- [Commands](./commands.md): every `nb` command line.
- [Configuration](./configuration/index.md): the three-tab configuration dialog.
  - [nexBash Options](./configuration/options.md): global toggles and rage reserves.
  - [Class Configuration](./configuration/class-configuration.md): attack and battlerage priority, profiles.
  - [Area Configuration](./configuration/area-configuration.md): per-area settings, target order, and NPC flags.

## Concepts

- [The decision model](./decision-model.md): how nexBash chooses what to do each
  prompt (lanes, gates, first-valid) — and where "rules" fit.
- [Strategies](./strategies.md): the per-class shipped baselines and customization.
- [Profiles](./profiles.md): named priority variants you can switch between.
- [Battlerage](./battlerage.md): the rage track, reserves, and coupling.
- [Areas](./areas.md): the area model, NPC flags, and discovery.
- [Session & automations](./session-and-automations.md): the run scoreboard,
  discovery, and the safety/effect automations.

For exact object shapes, callable names, topic names, and the persisted settings
